# 프로젝트 지침 (GEMINI.md)

## 1. 언어 및 커뮤니케이션
- **기본 언어:** 모든 응답과 커뮤니케이션은 항상 **한국어**로 진행한다.
- **응답 구성:**
  - 답변의 마지막에는 항상 해당 회차에서 수행한 작업을 요약하여 정리한다.
  - 마지막에는 향후 권장되는 다음 단계(추천 사항)를 몇 줄로 정리하여 제시한다.

## 2. 형상 관리 (Git)
- **원격 저장소:** `https://github.com/jjomingwa/webproduct-build-1`
- **배포 규칙:** 모든 변경사항은 작업이 완료된 후 최종적으로 `git push`를 진행하여 원격 저장소에 반영한다.

## 3. 코딩 원칙 (Coding Standards)
- **요구사항 준수:** 기존 요구사항을 누락하지 않도록 철저히 검토한다.
- **코드 일관성:** 코드가 복잡해지거나 깊어지더라도 논리적 구조, 모듈과 스타일이 일관되게 유지되도록 주의한다.
- **유기적 연결:** 새로운 기능이 추가될 때, 전체 시스템과의 상호작용 및 유기적 연결에 문제가 없는지 반드시 검토한다.
- **코드 추가(Add-on) 중심:** 코딩은 항상 기존 코드를 수정하기보다는 새로운 기능을 덧붙이고 "두껍게(Thickening)" 만드는 과정을 우선한다. (무분별한 삭제나 대규모 수정 지양)

## 4. 주석 및 문서화 (Documentation & Comments)
- **상세한 한국어 주석:** 모든 코드는 초보자도 이해할 수 있도록 상세한 한국어 주석을 포함해야 한다. (단순 기능 설명이 아닌, '왜' 이렇게 작성했는지에 대한 논리 포함)
- **파일별 헤더 설명:** 모든 코드 파일의 최상단에는 해당 파일의 목적, 구조, 작동 방식에 대한 상세한 설명을 작성한다.
- **코드 설명 전용 파일:** 프로젝트의 주요 로직과 구조를 한눈에 파악할 수 있는 `CODE_EXPLANATION.md` 파일을 유지 및 관리한다.

## 5. 업무 효율화 및 외부 도구 (Workflow Optimization)
- **Context7 MCP 활용:** 최신 라이브러리 정보나 API 문서가 필요한 경우, `smithery tool call context7-mcp query-docs`를 사용하여 항상 최신 정보를 참조한다. (AI 환각 현상 방지)
- **Smithery CLI:** 새로운 MCP 서버나 도구가 필요한 경우 Smithery 마켓플레이스를 통해 안전하게 연결하여 활용한다.
- **자동화된 Git 관리:** 모든 작업 완료 후 `git push`를 생활화하여 실시간 협업 및 백업 환경을 유지한다.
