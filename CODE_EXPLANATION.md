# 프로젝트 코드 설명서 (CODE_EXPLANATION.md)

이 파일은 프로젝트의 전체적인 구조, 각 모듈의 역할, 그리고 코드 내의 상세한 작동 원리를 초보자의 시각에서 설명하는 문서입니다.

## 1. 프로젝트 개요
- **목적:** 웹 프로젝트 구축을 위한 초기 템플릿 및 코딩 표준 정립
- **주요 기능:** (기능이 추가될 때마다 업데이트 예정)

## 2. 파일 및 폴더 구조
- `/`: 프로젝트 루트
  - `GEMINI.md`: 프로젝트 전체 지침 및 규칙
  - `CODE_EXPLANATION.md`: 현재 보고 계신 코드 상세 설명서
  - (추후 추가되는 파일들에 대한 설명이 여기에 명시됩니다.)

## 3. 코드 상세 설명 가이드라인
모든 코드 파일은 다음과 같은 기준을 따릅니다.

1. **파일 헤더:** 파일의 목적, 구조, 작동 방식 기술
2. **논리 기반 주석:** '무엇(What)'을 하는지뿐만 아니라 '왜(Why)' 그렇게 작성했는지 설명
3. **용어 설명:** 어려운 프로그래밍 용어나 라이브러리 함수에 대한 부연 설명 포함

## 4. 업무 효율화 도구 (Special Tools)

### Smithery CLI (@smithery/cli)
- **용도:** AI 에이전트 마켓플레이스인 Smithery를 통해 외부 도구(MCP 서버)를 쉽게 설치하고 관리합니다.
- **주요 명령어:**
  - `smithery login`: 인증 진행
  - `smithery mcp add <server>`: 새로운 도구 추가
  - `smithery tool list`: 연결된 도구 목록 확인

### Context7 MCP (upstash/context7-mcp)
- **용도:** 최신 버전의 API 문서와 코드 예제를 실시간으로 조회하여 AI가 정확한 답변을 내놓을 수 있도록 돕습니다.
- **핵심 가치:** 낡은 정보나 잘못된 코드를 생성할 확률을 현저히 낮추어 개발 생산성을 높입니다.
- **활용법:** 특정 라이브러리의 최신 스펙을 물어보거나 `query-docs` 도구를 통해 문서를 직접 주입합니다.
